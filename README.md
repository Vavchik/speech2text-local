# Speech2Text

Локальное приложение для транскрибации видео и аудио в текст.  
Работает полностью на вашем компьютере — данные никуда не отправляются.

---

## Возможности

- **Поддерживаемые форматы**: MP4, MKV, AVI, MOV, WebM, MP3, WAV, OGG, M4A, FLAC
- **Распознавание речи** через Whisper (GPU / CPU) — русский или английский
- **Нет ограничения по длительности** — обрабатывает файлы любой длины
- **Пакетная обработка** — можно выбрать несколько файлов, обработаются по очереди
- **Форматирование текста**: встроенное (заглавные буквы, точки) или через Ollama (пунктуация, абзацы)
- **Редактирование результата** — можно править текст прямо в окне перед сохранением
- **Сохранение настроек** — язык, модель, устройство запоминаются между запусками

## Быстрый старт

### 1. Установите Python 3.10–3.12
Скачайте с [python.org](https://www.python.org/downloads/). При установке поставьте галочку **«Add Python to PATH»**.

### 2. Установите FFmpeg
1. Скачайте [ffmpeg-release-essentials.zip](https://www.gyan.dev/ffmpeg/builds/)
2. Распакуйте, найдите папку `bin`
3. Добавьте её в системную переменную **PATH**
4. Проверьте: `ffmpeg -version`

### 3. Установите зависимости
Откройте терминал в папке проекта:

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 4. (Опционально) Установите CUDA для работы на видеокарте
- Проверьте: `nvidia-smi` — должна быть версия **CUDA 12.x**
- Если нет — скачайте [CUDA Toolkit 12.x](https://developer.nvidia.com/cuda-downloads)
- Если не хотите ставить CUDA — приложение будет работать на процессоре (`cpu`), но медленнее

## Запуск

### GUI (рекомендуется)

```bash
venv\Scripts\activate
python gui.py
```

Откроется окно:
1. Нажмите **«Обзор»** и выберите один или несколько файлов
2. Настройте параметры справа (язык, модель, устройство)
3. Нажмите **«Запустить»**
4. Когда обработка завершится, текст появится в нижней части окна
5. Текст можно редактировать, нажать **«Сохранить»**, **«Сохранить как...»** или **«Копировать»**

### CLI (для автоматизации)

```bash
venv\Scripts\activate
python transcribe.py video.mp4 --lang ru
```

#### Параметры

| Аргумент | Описание |
|----------|----------|
| `--lang en\|ru` | Язык (по умолчанию en) |
| `--mode raw\|formatted` | Сырой текст / с форматированием |
| `--output, -o FILE` | Куда сохранить результат |
| `--model MODEL` | Модель Whisper |
| `--device cuda\|cpu` | Устройство |

Пример:

```bash
python transcribe.py D:\видео\лекция.mp4 --lang ru --model large-v3-turbo
```

Результат сохраняется рядом с исходным файлом с расширением `.txt`.

## Модели Whisper

При первом запуске модель скачается автоматически в `~/.cache/whisper/`.

| Модель | Размер скачивания | VRAM | Скорость (RTX 4060) | Качество |
|--------|-------------------|------|---------------------|----------|
| **tiny** | ~150 MB | ~0.5 GB | 4–5x быстрее реального времени | Среднее |
| **base** | ~300 MB | ~1 GB | 2–3x | Хорошее |
| **small** | ~950 MB | ~1.5 GB | 1.5–2x | Очень хорошее |
| **medium** | ~3 GB | ~2.5 GB | 0.8–1.2x | Отличное |
| **large-v3-turbo** | ~3.5 GB | ~3.5 GB | 0.6–1x | Превосходное |

**Для видеокарт 4–6 GB VRAM** → `small` или `medium`.  
**Для 8+ GB VRAM** → `medium` или `large-v3-turbo`.  
**Без GPU (CPU)** → `tiny` или `base` (будет медленно в 5–10 раз).

## Форматирование текста (режим `formatted`)

В режиме `formatted` приложение улучшает текст: ставит точки и заглавные буквы, делит на предложения.

**Без Ollama** — работает встроенное простое форматирование:
```
было очень темно и страшно никто не хотел туда идти
→ Было очень темно и страшно. Никто не хотел туда идти.
```

**С Ollama** — более качественное форматирование: расстановка запятых, абзацы, исправление смысловых повторов.

### Настройка Ollama

1. Установите [Ollama](https://ollama.com)
2. Запустите: `ollama serve`
3. Скачайте модель: `ollama pull qwen2.5:7b-instruct` (или другую)
4. Откройте файл `.env` и укажите:
   ```
   OLLAMA_ENABLED=true
   OLLAMA_MODEL=qwen2.5:7b-instruct
   ```
5. В GUI выберите режим **formatted** или в CLI укажите `--mode formatted`

Если Ollama не запущена или модель недоступна — приложение автоматически использует встроенное форматирование.

## Настройки (.env)

Файл `.env` лежит в папке проекта. Можно редактировать в Блокноте.

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `WHISPER_MODEL` | medium | tiny / base / small / medium / large-v3-turbo |
| `WHISPER_DEVICE` | cuda | cuda / cpu |
| `WHISPER_COMPUTE_TYPE` | int8 | int8 / float16 |
| `CHUNK_DURATION_SEC` | 300 | Длина одного чанка в секундах |
| `CHUNK_OVERLAP_SEC` | 10 | Перекрытие между чанками |
| `OLLAMA_ENABLED` | false | Включить форматирование через Ollama |
| `OLLAMA_URL` | http://localhost:11434 | Адрес сервера Ollama |
| `OLLAMA_MODEL` | qwen2.5:7b-instruct | Модель для форматирования |

## Как это работает (пайплайн)

```
Видео / аудио файл
       ↓
FFmpeg → извлечение аудио (16 000 Гц, моно)
       ↓
Нарезка на чанки по 5 минут (+10 сек перекрытие для гладких стыков)
       ↓
faster-whisper → распознавание речи в каждом чанке (GPU)
       ↓
Склейка чанков → удаление дублирующихся слов на стыках
       ↓
(Опционально) Ollama → форматирование текста
       ↓
Сохранение .txt рядом с исходным файлом
```

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| `cublas64_12.dll not found` | Нет CUDA Toolkit. В GUI выберите `cpu` |
| `ffmpeg not found` | Установите FFmpeg и добавьте `bin` в PATH |
| Низкая скорость | Используйте `cuda` или модель поменьше |
| `out of memory` | Уменьшите модель: large → medium → small |
| Ошибка при форматировании | Проверьте что запущен `ollama serve` и модель существует |
| Текст кракозябрами | Откройте `.txt` через Notepad++ или VS Code (UTF-8) |
